FUNDAMENTAL PRINCIPLE

FireCalc V8 is a TOOL for fire safety decisions.

The RESPONSIBILITY for correct decisions belongs to:
  1. The Professional Engineer (PE)
  2. The Reviewing Authority Having Jurisdiction (AHJ)
  3. The Building Owner

NOT the software system.

PE ACCOUNTABILITY REQUIREMENTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

BEFORE an override is approved, the PE MUST:

1. INDEPENDENT CALCULATION
   ━━━━━━━━━━━━━━━━━━━
   ✓ PE performs hand calculations OR uses independent software
   ✓ PE verifies calculation methodology
   ✓ PE documents all assumptions
   
   The override is NOT based on FireCalc output alone.

2. PERSONAL REVIEW
   ━━━━━━━━━━━━━━━━━━━
   ✓ PE reviews actual building drawings
   ✓ PE visits site if necessary
   ✓ PE verifies all relevant code sections
   
   FireCalc patterns do not substitute for site review.

3. LEGAL SIGNATURE
   ━━━━━━━━━━━━━━━━━━━
   ✓ PE signs with their professional seal (digital or physical)
   ✓ PE's stamp appears on output document
   ✓ PE's license number is recorded
   
   The PE's stamp means:
   "I have independently verified this design."

4. DOCUMENTATION
   ━━━━━━━━━━━━━━━━━━━
   ✓ PE documents why manual override needed
   ✓ PE cites specific code sections
   ✓ PE notes any assumptions made
   
   This documentation becomes part of permanent record.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
OVERRIDE WORKFLOW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Standard Flow:
  1. FireCalc generates recommendation
  2. PE reviews output
  3. If agree: PE signs → Approved
  4. If disagree: → Override

Override Flow:
  1. PE identifies discrepancy
  2. PE performs independent calculation
  3. PE documents reasoning
  4. PE signs override with seal
  5. Override logged (encrypted audit trail)
  6. PE retains copy for liability record

✓ All override decisions they make
✓ Any design they certify
✓ Their professional seal on documents
