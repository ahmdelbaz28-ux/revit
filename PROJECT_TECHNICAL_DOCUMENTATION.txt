                        1. PROJECT OVERVIEW
================================================================================

PROJECT NAME: FireAI / FireSafetyGenius
PURPOSE: Fire safety analysis system for building drawings
TARGET USERS: Engineers, architects, fire safety consultants
JURISDICTION: NFPA 72, NEC 2023, ADA compliance

WHAT IT CLAIMS TO DO:
- Analyze architectural drawings for fire safety devices
- Calculate detector spacing per NFPA 72
- Optimize panel and device locations
- Check ADA mounting heights
- Run smoke simulation
- Learn from previous projects

WHAT IT ACTUALLY DOES:
- Advanced calculator with pattern matching
- Saves coordinates to database
- Applies mathematical formulas
- Runs simplified zone-based smoke model

CRITICAL NOTE: This is a TOOL, not an EXPERT.
           It calculates but does not UNDERSTAND.

                4. CRITICAL PROBLEMS IDENTIFIED
================================================================================

PROBLEM #1: ERROR AMPLIFICATION (CRITICAL - LIFE THREATENING)
========================================================================
ISSUE: The "learning" system can learn INCORRECT patterns from human errors.

DETAILS:
- Case-Based Reasoning (CBR) = memorizing, not understanding
- If engineer approves 10m spacing (violation), system records as "success"
- Future projects will repeat this error as "learned behavior"
- No truth validation before learning

EVIDENCE:
- Test: "Memorized coordinate" succeeds
- Test: "Applied lesson to new room shape" FAILS

RISK: Human errors get codified as "golden rules"
IMPACT: Could recommend non-compliant installations

FIX REQUIRED: Truth Filter Layer before any learning

--------------------------------------------------------------------------

PROBLEM #2: BLACK BOX REASONING (LEGAL LIABILITY)
========================================================================
ISSUE: System cannot explain decisions in court.

DETAILS:
- k-median says "panel here" but cannot explain WHY
- Output is just coordinates, no reasoning
- In court: "algorithm decided" = insufficient defense

EVIDENCE:
- No text explanation generation
- No rule tracing
- No audit trail with reasoning

RISK: Legal liability if installation fails
IMPACT: Cannot defend life-safety decisions

FIX REQUIRED: Decision Explanation (XAI) layer

--------------------------------------------------------------------------

PROBLEM #3: FALSE CONFIDENCE IN SIMULATION
========================================================================
ISSUE: Smoke simulation claims to be NFPA 92 but is simplified.

DETAILS:
- Zone Model runs in 17 seconds
- Real CFD (FDS) takes hours
- No field validation performed
- Results should not be used for critical decisions

EVIDENCE:
- Test: "Time to detect: 17s"
- Reality: Zone model not validated

RISK: Engineers may rely on inaccurate simulation
IMPACT: False sense of safety

STATUS: Documented but not fixed (requires CFD integration)

--------------------------------------------------------------------------

PROBLEM #4: MISSING ETHICAL CORE
========================================================================
ISSUE: System optimizes cost, not safety-first.

DETAILS:
- Will reduce devices if "code allows marginal compliance"
- No concept of "better for safety" vs "minimum code compliance"
- Can recommend undersizing to save money

EVIDENCE:
- Optimizer minimizes cable length
- No safety preference flag
- Cost optimization enabled by default

RISK: Life safety becomes secondary to cost
IMPACT: Cost-driven decisions over safety

FIX REQUIRED: Safety-first flag in all optimizers

================================================================================
                    5. KNOWN LIMITATIONS
================================================================================

LIMITATION #1: SELF-LEARNING IS NOT AI
------------------------------------------------------------------------
The "learning" system:
- Memorizes input-output pairs (CBR)
- Does NOT understand concepts
- Cannot generalize to new shapes
- Will repeat errors if recorded as "success"

STATUS: DOCUMENTED - Do not use for critical projects
RECOMMENDATION: Disable auto-learning until fixes implemented

--------------------------------------------------------------------------

LIMITATION #2: BEAM DETECTOR HANDLING
------------------------------------------------------------------------
Status: NOT IMPLEMENTED
Requires:
- Line-of-sight calculation
- Obstruction mapping
- 30.5m max spacing verification

RECOMMENDATION: Manual verification required

--------------------------------------------------------------------------

LIMITATION #3: HARDCODE CONSTANTS
------------------------------------------------------------------------
All spacing values are hardcoded:
- smoke_detector: 9.2m (should vary by edition)
- heat_detector: 15.2m
- pull_station: 61m travel distance

ISSUE: Different editions have different values
     Different jurisdictions use different editions

RECOMMENDATION: Database with edition citations needed

--------------------------------------------------------------------------

LIMITATION #4: FLOOR SEGMENTATION
------------------------------------------------------------------------
Status: NOT IMPLEMENTED
System cannot automatically identify:
- Room boundaries
- Smoke zones
- Occupancy types

RECOMMENDATION: Manual zone assignment required

--------------------------------------------------------------------------

LIMITATION #5: PDF QUALITY VARIANCE
------------------------------------------------------------------------
Extraction works on:
- Clean vector PDFs: ~95% accuracy
- Mixed vector/raster: ~70% accuracy
- Raster-only: ~50% accuracy

RECOMMENDATION: Legend verification required
